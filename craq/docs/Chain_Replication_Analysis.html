Abstract:
Chain replication is a new approach to coordinating clusters of fail-stop storage servers. The approach is intended for supporting large-scale storage services that exhibit high throughput and availability without sacrificing strong consistency guarantees. In chain replication, servers are organized as a chain. All requests that update objects are handled by the head of the chain and then propagated down the chain to the tail; all requests that query objects are handled by the tail. If the head fails, the next server in the chain becomes the new head; if the tail fails, the previous server in the chain becomes the new tail; if some other server fails, that server is simply removed from the chain. The approach supports a weak form of Byzantine failures through the use of hash trees constructed over the stored objects.
