Introduction (ç¬¬ä¸€æ®µ):
Large-scale storage services are becoming increasingly important as more applications migrate to network-based computing platforms. Such services must satisfy demanding requirements for throughput, availability, and consistency. Throughput must be high to support large numbers of clients. Availability must be high so that clients are not blocked by server failures. And consistency must be strong so that clients observe behavior that is intuitive and predictable.
Introduction (ç¬¬äºŒæ®µ):
Unfortunately, there are fundamental tensions between these requirements. Protocols that provide strong consistency typically require coordination among servers, and this coordination can limit throughput and availability. For example, protocols based on state machine replication require that a majority of servers agree before any request can be processed, which can be slow when servers are geographically distributed or when network partitions occur.
Introduction (ç¬¬ä¸‰æ®µ):
Chain replication is a new approach to coordinating replicated storage servers that is intended to support large-scale storage services. The approach organizes servers in a chain and uses a simple protocol to coordinate updates. All update requests are sent to the head of the chain and propagated to the tail; all query requests are sent to the tail. This organization has several advantages: (i) it provides strong consistency, (ii) it can achieve high throughput for query-intensive workloads, and (iii) it simplifies the implementation of many storage service features.
